1. Multi-Head Attention (MHA) 回顾

标准的多头注意力：ddd 表示嵌入维度，nhn_hnh​ 表示注意力头的数量，dhd_hdh​ 表示每个头的维度，ht∈Rdh_t \in R^dht​∈Rd 表示在注意力层中第t个token的注意力输入。

标准MHA机制通过三个矩阵WQ、WK、WV∈Rdhnh×dW^Q、W^K、W^V \in \mathbb{R}^{d_h n_h \times d}WQ、WK、WV∈Rdh​nh​×d分别生成输入hth_tht​的查询向量qt∈R𝑑hq_t\in \mathbb{R}^{𝑑_ℎ}qt​∈Rdh​、键向量kt∈R𝑑hk_t\in \mathbb{R}^{𝑑_ℎ}kt​∈Rdh​和值向量vt∈R𝑑hv_t\in \mathbb{R}^{𝑑_ℎ}vt​∈Rdh​。公式如下：

qt=WQht(1)q_t = W^Q h_t \tag{1} qt​=WQht​(1)

kt=WKht(2)k_t = W^K h_t \tag 2 kt​=WKht​(2)

vt=WVht(3)v_t = W^V h_t \tag 3 vt​=WVht​(3)

接下来，qtq_tqt​，ktk_tkt​，vtv_tvt​将被切分成nhn_hnh​个头，用于多头注意力计算:

[qt,1;qt,2;…;qt,nh]=qt(4)[q_{t,1}; q_{t,2}; \ldots; q_{t,n_h}] = q_t \quad \tag 4 [qt,1​;qt,2​;…;qt,nh​​]=qt​(4)

[𝑘𝑡,1;𝑘𝑡,2;…;𝑘𝑡,𝑛h]=𝑘𝑡(5)[𝑘_{𝑡,1}; 𝑘_{𝑡,2}; \ldots; 𝑘_{𝑡,𝑛_ℎ}] = 𝑘_𝑡 \quad \tag 5 [kt,1​;kt,2​;…;kt,nh​​]=kt​(5)

[𝑣𝑡,1;𝑣𝑡,2;…;𝑣𝑡,𝑛h]=𝑣𝑡(6)[𝑣_{𝑡,1}; 𝑣_{𝑡,2}; \ldots; 𝑣_{𝑡,𝑛_ℎ}] = 𝑣_𝑡 \quad \tag 6 [vt,1​;vt,2​;…;vt,nh​​]=vt​(6)

其中：

  • qt,i,kt,i,vt,i∈Rdhq_{t,i}, k_{t,i}, v_{t,i} \in \mathbb{R}^{d_h}qt,i​,kt,i​,vt,i​∈Rdh​ 是第 ttt 个 token 的第i个头的 query、key、value 向量。
  • nhn_hnh​ 是注意力头数，QKV分头前的特征维度为ddd，分头后每个头的维度是 dhd_hdh​。

公式含义：将原始的 qt,kt,vtq_t, k_t, v_tqt​,kt​,vt​ 按列切分成 nhn_hnh​ 个头的子向量。

每个头的注意力计算公式为:

attention_weightt,i=Softmaxj=1t(qt,i⊤kj,idh)(7.1)\text{attention\_weight}_{t,i} = \text{Softmax}_{j=1}^{t}\left( \frac{q_{t,i}^\top k_{j,i}}{\sqrt {d_h}} \right) \tag {7.1} attention_weightt,i​=Softmaxj=1t​(dh​​qt,i⊤​kj,i​​)(7.1)

最后拼接所有头的输出，再做输出投影:

𝑢𝑡=𝑊𝑂[𝑜𝑡,1;𝑜𝑡,2;…;𝑜𝑡,𝑛h](8)𝑢_𝑡 = 𝑊^𝑂 [𝑜_{𝑡,1}; 𝑜_{𝑡,2}; \ldots; 𝑜_{𝑡,𝑛_ℎ}] \tag 8 ut​=WO[ot,1​;ot,2​;…;ot,nh​​](8)

其中 𝑊𝑂∈R𝑑×𝑑h𝑛h𝑊^𝑂 \in \mathbb{R}^{𝑑 \times 𝑑_ℎ 𝑛_ℎ}WO∈Rd×dh​nh​ 是输出映射矩阵。

推理时问题：

  • 所有 key 和 value 需要缓存，KV cache大小为 2𝑛h𝑑h𝑙2𝑛_ℎ 𝑑_ℎ 𝑙2nh​dh​l（𝑙𝑙l 为序列长度），当 batch size 和 𝑙𝑙l 很大时会占用大量显存。

2. Low-Rank Key-Value Joint Compression（低秩 KV 联合压缩）

MLA 核心是通过低秩联合压缩减少 KV cache大小。

  1. 压缩输入:

    将输入 $ h_t $ 压缩到KV低维共享表示 $ c_t^{KV} $：

    𝑐𝑡𝐾𝑉=𝑊𝐷𝐾𝑉h𝑡(9)𝑐_{𝑡}^{𝐾𝑉} = 𝑊^{𝐷𝐾𝑉} ℎ_𝑡 \tag 9 ctKV​=WDKVht​(9)

    • 𝑐𝑡𝐾𝑉∈R𝑑𝑐𝑐_{𝑡}^{𝐾𝑉} \in \mathbb{R}^{𝑑_𝑐}ctKV​∈Rdc​：压缩后的 latent 表示，𝑑𝑐≪𝑑h𝑛h𝑑_𝑐 \ll 𝑑_ℎ 𝑛_ℎdc​≪dh​nh​。
    • 𝑊𝐷𝐾𝑉∈R𝑑𝑐×𝑑𝑊^{𝐷𝐾𝑉} \in \mathbb{R}^{𝑑_𝑐 \times 𝑑}WDKV∈Rdc​×d：降维矩阵，D的含义是降维down_sample。

  2. 解码 key 和 value:

    从 𝑐𝑡𝐾𝑉𝑐_{𝑡}^{𝐾𝑉}ctKV​ 解码出 key 和 value：

    𝑘𝑡𝐶=𝑊𝑈𝐾𝑐𝑡𝐾𝑉(10)𝑘_{𝑡}^{𝐶} = 𝑊^{𝑈𝐾} 𝑐_{𝑡}^{𝐾𝑉} \tag {10} ktC​=WUKctKV​(10)

    𝑣𝑡𝐶=𝑊𝑈𝑉𝑐𝑡𝐾𝑉(11)𝑣_{𝑡}^{𝐶} = 𝑊^{𝑈𝑉} 𝑐_{𝑡}^{𝐾𝑉} \tag {11} vtC​=WUVctKV​(11)

    • 𝑊𝑈𝐾,𝑊𝑈𝑉∈R𝑑h𝑛h×𝑑𝑐𝑊^{𝑈𝐾}, 𝑊^{𝑈𝑉} \in \mathbb{R}^{𝑑_ℎ 𝑛_ℎ \times 𝑑_𝑐}WUK,WUV∈Rdh​nh​×dc​：升维矩阵，U的含义是升维up_sample。

  3. 低秩压缩后的注意力权重计算方式：

𝑞𝑡⊤𝑘j𝐶=(WQht)⊤𝑊𝑈𝐾𝑐𝑡𝐾𝑉=ht⊤(WQ)⊤𝑊𝑈𝐾𝑐𝑡𝐾𝑉=ht(𝑊𝑈𝐾)⊤WQ𝑐𝑡𝐾𝑉𝑞_{𝑡}^\top 𝑘_{j}^{𝐶} = (W^Q h_t)^\top 𝑊^{𝑈𝐾} 𝑐_{𝑡}^{𝐾𝑉} = h_t^\top (W^Q)^\top 𝑊^{𝑈𝐾} 𝑐_{𝑡}^{𝐾𝑉} = \\ h_t (𝑊^{𝑈𝐾})^\top W^Q 𝑐_{𝑡}^{𝐾𝑉} qt⊤​kjC​=(WQht​)⊤WUKctKV​=ht⊤​(WQ)⊤WUKctKV​=ht​(WUK)⊤WQctKV​

  1. 最后的输出:

多头注意力相当于把 注意力权重attention_weightattention\_weightattention_weight从一维向量变为了二维向量，shape: (seq_len,) -> (seq_len, head_num)，但本质上还是一个张量，只是得到这个张量的方式注意力机制计算比较复杂。

[𝑜𝑡,1;𝑜𝑡,2;…;𝑜𝑡,𝑛h]=vt@attention_weight=𝑊𝑈𝑉𝑐𝑡𝐾𝑉@attention_weight[𝑜_{𝑡,1}; 𝑜_{𝑡,2}; \ldots; 𝑜_{𝑡,𝑛_ℎ}] = v_t @ attention\_weight = \\ 𝑊^{𝑈𝑉} 𝑐_{𝑡}^{𝐾𝑉} @ attention\_weight [ot,1​;ot,2​;…;ot,nh​​]=vt​@attention_weight=WUVctKV​@attention_weight

𝑢𝑡=𝑊𝑂[𝑜𝑡,1;𝑜𝑡,2;…;𝑜𝑡,𝑛h]𝑢𝑡=𝑊𝑂𝑊𝑈𝑉(𝑐𝑡𝐾𝑉@attention_weight)𝑢_𝑡 = 𝑊^𝑂 [𝑜_{𝑡,1}; 𝑜_{𝑡,2}; \ldots; 𝑜_{𝑡,𝑛_ℎ}] \\ 𝑢_𝑡 = 𝑊^𝑂 𝑊^{𝑈𝑉} (𝑐_{𝑡}^{𝐾𝑉} @ attention\_weight) ut​=WO[ot,1​;ot,2​;…;ot,nh​​]ut​=WOWUV(ctKV​@attention_weight)

优势：

  • 推理时仅需缓存 𝑐𝑡𝐾𝑉𝑐_{𝑡}^{𝐾𝑉}ctKV​（每 token 只需 𝑑𝑐𝑑_𝑐dc​ 维），KV 缓存大小从 2𝑑h𝑛h𝑙2𝑑_ℎ 𝑛_ℎ 𝑙2dh​nh​l 缩减到 𝑑𝑐𝑙𝑑_𝑐 𝑙dc​l。
  • 可将 𝑊𝑈𝐾∈R𝑑h𝑛h×𝑑𝑐𝑊^{𝑈𝐾} \in \mathbb{R}^{𝑑_ℎ 𝑛_ℎ \times 𝑑_𝑐}WUK∈Rdh​nh​×dc​ 与 𝑊𝑄∈Rdhnh×d𝑊^𝑄 \in \mathbb{R}^{d_h n_h \times d}WQ∈Rdh​nh​×d合并，将𝑊𝑈𝑉∈R𝑑h𝑛h×𝑑𝑐𝑊^{𝑈𝑉} \in \mathbb{R}^{𝑑_ℎ 𝑛_ℎ \times 𝑑_𝑐}WUV∈Rdh​nh​×dc​ 与 𝑊𝑂∈R𝑑×𝑑h𝑛h𝑊^𝑂 \in \mathbb{R}^{𝑑 \times 𝑑_ℎ 𝑛_ℎ}WO∈Rd×dh​nh​合并，无需显式生成或存储 key/value。

矩阵合并分析：

上面公式看似需要使用𝑐𝑡𝐾𝑉𝑐_{𝑡}^{𝐾𝑉}ctKV​重新生成历史的KV，但是通过权重吸收(合并)可以将重新生成的步骤合并其他向量的投影中，从而无需真正重新生成。

𝑊𝑈𝐾∈R𝑑h𝑛h×𝑑𝑐𝑊^{𝑈𝐾} \in \mathbb{R}^{𝑑_ℎ 𝑛_ℎ \times 𝑑_𝑐}WUK∈Rdh​nh​×dc​ 与 𝑊𝑄∈Rdhnh×d𝑊^𝑄 \in \mathbb{R}^{d_h n_h \times d}WQ∈Rdh​nh​×d的合并，维度变换是 d−>dhnh−>dcd -> d_hn_h -> d_cd−>dh​nh​−>dc​，吸收后是 d−>dcd -> d_cd−>dc​，如果是原始的MHA不进行维度压缩d−>dhnhd -> d_hn_hd−>dh​nh​，可以看出来吸收后相比MHA计算量减少，不吸收则增大计算量。

𝑊𝑈𝑉∈R𝑑h𝑛h×𝑑𝑐𝑊^{𝑈𝑉} \in \mathbb{R}^{𝑑_ℎ 𝑛_ℎ \times 𝑑_𝑐}WUV∈Rdh​nh​×dc​ 与 𝑊𝑂∈R𝑑×𝑑h𝑛h𝑊^𝑂 \in \mathbb{R}^{𝑑 \times 𝑑_ℎ 𝑛_ℎ}WO∈Rd×dh​nh​合并，维度变换是 dc−>dhnh−>dd_c -> d_hn_h -> ddc​−>dh​nh​−>d, 吸收后是dc−>dd_c -> ddc​−>d，同样吸收后相比MHA计算量减少，不吸收则增大计算量。

3. Low-Rank Query Compression（低秩 Query 压缩）

虽然压缩 Query 不能减少 KV 缓存，但是也能节省计算。

  1. 压缩 Query：

    𝑐𝑡𝑄=𝑊𝐷𝑄h𝑡(12)𝑐_{𝑡}^{𝑄} = 𝑊^{𝐷𝑄} ℎ_𝑡 \quad \tag {12} ctQ​=WDQht​(12)

    𝑞𝑡𝐶=𝑊𝑈𝑄𝑐𝑡𝑄(13)𝑞_{𝑡}^{𝐶} = 𝑊^{𝑈𝑄} 𝑐_{𝑡}^{𝑄} \quad \tag {13} qtC​=WUQctQ​(13)

    • 𝑐𝑡𝑄∈R𝑑𝑐′𝑐_{𝑡}^{𝑄} \in \mathbb{R}^{𝑑_𝑐'}ctQ​∈Rdc′​：压缩后的 query 表示，𝑑𝑐′≪𝑑h𝑛h𝑑_𝑐' \ll 𝑑_ℎ 𝑛_ℎdc′​≪dh​nh​。
    • 𝑊𝐷𝑄∈R𝑑𝑐′×𝑑𝑊^{𝐷𝑄} \in \mathbb{R}^{𝑑_𝑐' \times 𝑑}WDQ∈Rdc′​×d，𝑊𝑈𝑄∈R𝑑h𝑛h×𝑑𝑐′𝑊^{𝑈𝑄} \in \mathbb{R}^{𝑑_ℎ 𝑛_ℎ \times 𝑑_𝑐'}WUQ∈Rdh​nh​×dc′​：降维和升维矩阵。

作用：

  • 降维后再升维，减少全连接计算量，尤其适用于输入维度 𝑑𝑑d 较大的场景。

4. Decoupled Rotary Position Embedding（解耦 RoPE）

背景

前面的推理没有考虑位置编码，如果考虑RoPE位置编码，会发现权重吸收和位置编码不兼容：

𝑞𝑡⊤𝑘j𝐶=>RoPE(𝑞𝑡⊤)RoPE(𝑘j𝐶)(-1)𝑞_{𝑡}^\top 𝑘_{j}^{𝐶} => RoPE(𝑞_{𝑡}^\top)RoPE(𝑘_{j}^{𝐶}) \tag {-1} qt⊤​kjC​=>RoPE(qt⊤​)RoPE(kjC​)(-1)

RoPE(𝑞𝑡⊤)RoPE(𝑘j𝐶)=(Rt𝑞𝑡)⊤Rj𝑘j𝐶=𝑞𝑡⊤Rt⊤Rj𝑘j𝐶=𝑞𝑡⊤Rj−t𝑘j𝐶=ht⊤(WQ)⊤Rj−t𝑊𝑈𝐾𝑐𝑡𝐾𝑉(-2)\begin{array}{c} RoPE(𝑞_{𝑡}^\top)RoPE(𝑘_{j}^{𝐶}) = (R_t𝑞_{𝑡})^\top R_j 𝑘_{j}^{𝐶} = \\ 𝑞_{𝑡}^\top R_t^\top R_j 𝑘_{j}^{𝐶} = 𝑞_{𝑡}^\top R_{j-t} 𝑘_{j}^{𝐶} = h_t^\top (W^Q)^\top R_{j-t} 𝑊^{𝑈𝐾} 𝑐_{𝑡}^{𝐾𝑉} \tag {-2} \end{array} RoPE(qt⊤​)RoPE(kjC​)=(Rt​qt​)⊤Rj​kjC​=qt⊤​Rt⊤​Rj​kjC​=qt⊤​Rj−t​kjC​=ht⊤​(WQ)⊤Rj−t​WUKctKV​​(-2)

前面 𝑊𝑈𝐾𝑊^{𝑈𝐾}WUK 与 𝑊𝑄𝑊^𝑄WQ 可以合并，是因为(WQ)⊤𝑊𝑈𝐾(W^Q)^\top 𝑊^{𝑈𝐾}(WQ)⊤WUK 是常量，但是(WQ)⊤Rj−t𝑊𝑈𝐾(W^Q)^\top R_{j-t} 𝑊^{𝑈𝐾}(WQ)⊤Rj−t​WUK是与query向量的位置ttt相关的变量Rj−tR_{j-t}Rj−t​，即随推理过程中query的位置变化而变化，导致旧的缓存不能直接使用，所以无法直接合并。

为什么GQA的低秩投影没有受RoPE影响，因为GQA只减少了头数，计算时又恢复了头数，但是隐式恢复头数直接通过广播实现，不涉及特征维度改变序列长度改变；而MLA减小的是每个头的特征维度，计算时为了避免增加计算量，只能隐式恢复特征维度，即需要借助权重吸收，而两个权重之间又夹着RoPE，没办法权重吸收。

RoPE（Rotary Position Embedding）对 key 和 query 都是位置敏感的。

若直接将 RoPE 应用于压缩后的 key，升维矩阵 𝑊𝑈𝐾𝑊^{𝑈𝐾}WUK 无法与 𝑊𝑄𝑊^𝑄WQ 合并，影响推理效率？

一个简单的逻辑应该是把位置编码都应用到压缩后的特征上。

原因

  • RoPE 是依赖位置的旋转矩阵，矩阵乘法不满足交换律。
  • 若在升维后的 key 上应用 RoPE，需重新计算所有历史 key 的位置编码，无法仅用缓存恢复。

解决方案

将 query 和 key 分为两部分，一部分就用MLA不使用RoPE 位置编码，另一部分用MQA使用位置编码：

  1. 对 query 和 key 分别应用 RoPE：

    将输入的查询内容向量 ctQc^Q_tctQ​通过矩阵 WQRW^{QR}WQR投影到 RoPE 编码后的空间，得到分离后的旋转位置编码查询向量 qtRq^R_tqtR​，并按多头nhn_hnh​分块。

    [𝑞𝑡,1𝑅;𝑞𝑡,2𝑅;…;𝑞𝑡,𝑛h𝑅]=RoPE(𝑊𝑄𝑅𝑐𝑡𝑄)(14)[𝑞_{𝑡,1}^𝑅; 𝑞_{𝑡,2}^𝑅; \ldots; 𝑞_{𝑡,𝑛_ℎ}^𝑅] = \text{RoPE}(𝑊^{𝑄𝑅} 𝑐_{𝑡}^{𝑄}) \quad \text{(14)} [qt,1R​;qt,2R​;…;qt,nh​R​]=RoPE(WQRctQ​)(14)

    将隐藏状态 $ h_t $ 通过矩阵 WKRW^{KR}WKR投影，并应用 RoPE 得到分离后的旋转位置编码键向量$ k^R_t $。

    𝑘𝑡𝑅=RoPE(𝑊𝐾𝑅h𝑡)(15)𝑘_{𝑡}^𝑅 = \text{RoPE}(𝑊^{𝐾𝑅} ℎ_𝑡) \quad \text{(15)} ktR​=RoPE(WKRht​)(15)

  2. 拼接压缩部分和 RoPE 部分：

    将内容查询向量与旋转位置编码查询向量进行拼接，得到最终的第 iii个头的查询向量。

5. 总结 MLA 思路

MLA 的核心目标是降低推理时的显存占用和计算延迟，同时保持注意力效果。具体方法包括：

  1. 低秩联合压缩 KV：通过共享压缩表示 𝑐𝑡𝐾𝑉𝑐_{𝑡}^{𝐾𝑉}ctKV​ 减少 KV 缓存大小。
  2. 低秩 Query 压缩：减少 Query 的计算量。
  3. 解耦 RoPE：在保留位置编码效果的同时，使压缩策略兼容 RoPE，避免破坏缓存复用。

  • 计算效率提升，适用于大规模模型部署。
